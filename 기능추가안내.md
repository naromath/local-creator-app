# 사업계획서 관리 기능 — 추가 구현 안내

## 📌 새로 추가된 기능

### 1. 사업계획서 파일 관리
신규 지원자가 선정되면 **사업계획서를 온라인으로 업로드**할 수 있습니다.

**위치**:
- 창업기업 목록 (`/companies`) → 각 기업의 **"📄 업로드"** 버튼
- 또는 기업 상세 (`/companies/[id]`) → **"사업계획서"** 탭

---

## 🚀 사용 방법

### 기업 등록 후 사업계획서 업로드

#### **Step 1: 기업 등록**
1. 창업기업 관리 페이지에서 **"+ 신규 등록"** 클릭
2. 신규 지원자 정보 입력 및 등록
3. 기업이 목록에 추가됨

#### **Step 2: 사업계획서 업로드**
**방법 A: 기업 목록에서 직접 업로드**
1. 등록된 기업 행의 **"📄 업로드"** 버튼 클릭
2. 파일 선택 또는 드래그
3. 비고 입력 (선택)
4. **"업로드"** 클릭

**방법 B: 기업 상세에서 업로드**
1. 기업명을 클릭하여 상세 페이지 접속
2. **"사업계획서"** 탭 클릭
3. **"파일 업로드"** 버튼 클릭
4. 파일 업로드 진행

#### **Step 3: 파일 관리**
- **다운로드**: "다운로드" 버튼으로 파일 다운로드
- **삭제**: "삭제" 버튼으로 파일 제거 (확인 후 삭제)
- **조회**: 사업계획서 탭에서 모든 업로드된 파일 확인

---

## 📋 지원 파일 형식

| 형식 | 확장자 | 용도 |
|------|--------|------|
| PDF | `.pdf` | 사업계획서, 문서 |
| Word | `.doc, .docx` | 사업계획서, 제안서 |
| Excel | `.xls, .xlsx` | 재무계획, 예산표 |
| PowerPoint | `.ppt, .pptx` | 발표 자료 |

**제한사항**:
- 최대 파일 크기: **50MB**
- 한 기업당 **여러 개 파일** 업로드 가능
- 파일명, 업로드 날짜, 크기 자동 기록

## 💾 데이터 저장

### 데이터베이스 테이블: `business_plans`

| 필드 | 설명 |
|------|------|
| `id` | 파일 고유 ID |
| `company_id` | 기업 ID (외래키) |
| `file_name` | 원본 파일명 |
| `file_path` | Supabase Storage 경로 |
| `file_size` | 파일 크기 (바이트) |
| `file_type` | 파일 확장자 |
| `uploaded_by` | 업로드자 (기본값: "관리자") |
| `uploaded_at` | 업로드 시간 |
| `notes` | 비고 (선택사항) |
| `created_at` | 생성 시간 |
| `updated_at` | 수정 시간 |

### 파일 저장소: Supabase Storage

```
business-plans/
  └── {company_id}/
      └── {timestamp}-{filename}
          예: business-plans/5/1711474800000-2025년_사업계획서.pdf
```

## 🔧 기술 사항

### 추가된 컴포넌트
- **`FileUploadModal.tsx`**: 파일 업로드 모달 컴포넌트
  - 파일 타입 검증 (PDF, Word, Excel, PowerPoint)
  - 파일 크기 검증 (최대 50MB)
  - 업로드 진행률 표시

### 수정된 페이지
1. **`/companies` 페이지**
   - 기업 목록에 "📄 업로드" 버튼 추가
   - 테이블에 사업계획서 열 추가

2. **`/companies/[id]` 페이지**
   - "사업계획서" 탭 추가
   - 파일 목록 조회/다운로드/삭제 기능
   - 업로드 모달 통합

### 새 데이터베이스 테이블
- `business_plans`: 사업계획서 파일 메타데이터 저장

## 🔐 보안

- **인증**: Supabase Auth로 로그인한 관리자만 접근 가능
- **파일 타입 검증**: 업로드 전 파일 형식 확인
- **파일 크기 제한**: 50MB 이상 파일 거부
- **저장소 보안**: Supabase Storage의 기본 RLS 정책 적용

## 📝 예시 워크플로우

```
1. "신규 기업 A" 등록
   ↓
2. 기업 목록에서 "📄 업로드" 클릭
   ↓
3. "2025년_사업계획서_최종.pdf" 파일 선택
   ↓
4. 비고: "대표가 최종 승인함" 입력
   ↓

   ↓
6. 기업 상세 → "사업계획서" 탭에서 파일 확인
   ↓
7. "다운로드"로 검토 가능 또는 "삭제"로 제거 가능
```

## ⚠️ 주의사항

1. **파일 형식**: 지원하는 형식만 업로드 가능
2. **파일 크기**: 50MB 초과 시 업로드 불가
3. **파일명**: 특수문자는 자동으로 처리됨
4. **삭제 확인**: 삭제 전 확인 메시지가 표시됨
5. **다중 업로드**: 같은 기업에 여러 버전 업로드 가능

## 📞 문제 해결

**파일 업로드 실패**
- 파일 형식이 지원되는 형식인지 확인
- 파일 크기가 50MB 이하인지 확인
- 네트워크 연결 상태 확인

**파일이 조회되지 않음**
- 페이지 새로고침 (F5)
- 기업이 올바르게 저장되었는지 확인
- 브라우저 캐시 삭제

**다운로드 링크 오류**
- Supabase Storage 버킷 권한 확인
- Storage의 public 설정 확인
